# INTERVENTION CARD

## Card Overview

| Attribute | Value |
|-----------|-------|
| **Type** | Core Card |
| **Color** | Orange (#FFB347) |
| **Size** | 5" × 7" (or 8.5" × 11" for complex cases) |
| **Paper** | 80-100 lb cardstock, matte |
| **Used By** | Scribe at Station 4 (during provider encounter) |
| **Timing** | Minutes 21-32 of 40-minute visit |

## Purpose

Document provider clinical decisions, treatment plan, and orders. This card captures the "plan" that drives patient care forward. Designed for real-time scribe documentation during the provider encounter.

### Data Fields

| Section | Fields |
|---------|--------|
| **Assessment** | Primary diagnosis, CKD Stage, Secondary diagnoses (up to 3) |
| **GDMT Status** | ACEi/ARB current/action, SGLT2i current/action, MRA current/action, contraindications |
| **Medication Changes** | Started, Stopped, Dose changed, No changes checkbox |
| **Labs Ordered** | Checkboxes for common labs + other field, timing |
| **Referrals** | Transplant center, Vascular access, Dietitian, Diabetes Ed, Cardiology, Other |
| **Follow-up** | Timeframe selection |
| **Discussion Points** | 3 key points for patient summary |
| **Signatures** | Provider signature + Date, Scribe initials |

### GDMT Quick Reference

Printed on card back or separate quick reference:

```
ACEi/ARB Decision:
├─ On max dose → Continue
├─ On submaximal → Consider increase if BP/proteinuria allow
├─ Not on → Start if no contraindication
└─ Contraindicated → Document reason (hyperkalemia, bilateral RAS, pregnancy)

SGLT2i Decision:
├─ On → Continue
├─ Not on, GFR >20 → Start (preferred: Jardiance 10mg, Farxiga 10mg)
├─ Not on, GFR <20 → May continue if already on, avoid initiation
└─ Contraindicated → Document reason (frequent UTI, DKA history, Type 1 DM)

MRA Decision:
├─ On → Continue, monitor K+
├─ Not on, K+ <5.0 → Consider starting (Spironolactone 25mg or Finerenone)
└─ K+ ≥5.0 → Hold, reassess after K+ management
```

## Training Notes

**For Scribes:**
1. Document in real-time during provider encounter
2. Capture exact medication names and doses
3. Ensure GDMT status is explicitly addressed every visit
4. Discussion Points should be in plain language for patient handoff
5. Get provider signature before they leave the pod

**For Providers:**
1. Verbalize decisions clearly for scribe documentation
2. Address GDMT status systematically
3. Provide 3 key discussion points in patient-friendly language
4. Review and sign card before moving to next patient

**Common Errors to Avoid:**
- Leaving GDMT boxes unchecked (must document current status and action)
- Not specifying lab timing
- Vague discussion points (should be specific and actionable)
- Missing provider signature
