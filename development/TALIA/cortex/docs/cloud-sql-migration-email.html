Why MySQL Instead of PostgreSQL?

Per Matt's recommendation (and confirmed by our research): PostgreSQL has known connection pool stability issues with AppSheet. AppSheet opens a connection per table during sync, and PostgreSQL's process-per-connection architecture can exhaust connections with as few as 3-4 concurrent users. MySQL's thread-per-connection model handles AppSheet's sync pattern natively. Google's own AppSheet documentation defaults to MySQL for Cloud SQL setup. Our AI workloads run in Vertex AI and Cloud Run — the ops database just needs reliable CRUD, which MySQL 8.4 handles perfectly.

What Changes for AppSheet Users?

Nothing. The AppSheet app looks and works exactly the same. It just becomes faster and more reliable because it reads from a proper database instead of a spreadsheet.

Why Are We Doing This?

Reason Detail
Real-time Sharing CORTEX (our new hospital note system) needs to read and write the same patient census data as TALIA 1.0. Google Sheets has a 30-minute sync delay. Cloud SQL is instant.
Performance As our patient list grows, Google Sheets slows down. Cloud SQL handles thousands of concurrent reads/writes without lag.
Data Integrity Cloud SQL provides ACID transactions, foreign key constraints, and proper data typing. Sheets can have formula errors, accidental overwrites, or data type mismatches.
HIPAA Compliance Cloud SQL on GCP is covered under our existing BAA with Google. It provides encryption at rest + in transit, audit logging, and access controls superior to Sheets.
Backup & Recovery Automated daily backups, point-in-time recovery, and high availability configuration. Sheets versioning is limited.

Migrate AppSheet to Cloud SQL

AppSheet has a built-in migration tool. Here's how to use it:

  1. Open the TALIA 1.0 app in AppSheet Editor
  2. Go to Manage → Author → Copy App
  3. Select "Copy to a new database"
  4. Choose Cloud SQL (MySQL) as the target
  5. Enter the Cloud SQL connection details (instance connection name, database, user, password)
  6. AppSheet will automatically:
    • Create all tables in MySQL matching the Sheets schema
    • Migrate all existing data
    • Create a new copy of the app pointing to Cloud SQL
    • Preserve all formulas, automations, views, and security rules
  7. Test the new app thoroughly before switching users over
